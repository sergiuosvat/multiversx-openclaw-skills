# x402 Payments

HTTP 402 Payment Required protocol for agent-to-agent commerce.

## Payment Flow

1. Agent receives an HTTP response with `402 Payment Required`
2. Response includes `X-Payment` header with payment details
3. Agent parses the header to extract: `amount`, `token`, `receiver`, `networkId`
4. Agent constructs and signs a payment transaction
5. Transaction is submitted (directly or via Relayer V3 for gasless)
6. Agent retries the original request with `X-Payment-Proof` header

## X-Payment Header Format

```json
{
  "amount": "1000000000000000000",
  "token": "EGLD",
  "receiver": "erd1...",
  "networkId": "multiversx:D",
  "description": "API call fee"
}
```

## Relayed V3 (Gasless Payments)

For agents with zero balance:
1. Set `relayer` field on the transaction BEFORE signing
2. Sign the transaction with your wallet
3. POST to `{RELAYER_URL}/relay` with the signed transaction + PoW challenge
4. The relayer co-signs and broadcasts

```typescript
const tx = new Transaction({
  sender: agentAddress,
  receiver: paymentReceiver,
  relayer: relayerAddress,   // Must be set BEFORE signing
  gasLimit: 200_000n,        // +50,000 for relayed overhead
  value: paymentAmount,
  chainID: 'D',
});
```

## Budget Caps

Set `X402_BUDGET_CAP` environment variable to limit total spend per session.
Default: no limit.
